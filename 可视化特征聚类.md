# 可视化特征聚类实验

> 验证假说：经过对比学习微调的 projector 是否在特征空间里建立了**跨物体类别的通用异常先验**。

---

## 一、动机

### 1.1 核心观察：位姿不对齐时仍能正确高亮异常

训练阶段，每个三元组 `(N1, N2, A1)` 是**同物体同位姿**，三张图共享同一组随机
crop / flip 参数：

```
N1, N2:  同物体、不同光照下的正常图，pixel-level 完美对齐
A1:      同物体的真实异常图，对应位置相同
```

训练目标是在**同一像素位置**上拉近 N1↔N2、推远 N1↔A1。也就是说，**模型从未见过
"位置不对齐的正常 patch 对"**——训练 loss 只对齐位置上的余弦相似度。

但在 testdata 推理时，ref 和 query，**经常存在轻微位姿偏差**。
在这种情况下：

- 同位置 patch 上的 ref 和 query 内容不再完全对应（一个正常区域可能和另一个正常区域比较）
- 按训练目标推理，应该会产生大量"假异常"分数（因为两个不同位置的正常 patch 本来就不像）

但实测可视化结果显示：**即便位姿有偏差，模型仍任然在缺陷区域高亮**。

**这意味着**：

```
cos( normal_patch_at_pos_A,  normal_patch_at_pos_B )         (位置不对齐造成的不相似)
>
cos( normal_patch_at_pos_X,  anomalous_patch_at_pos_X )      (异常造成的不相似)
```

也就是说，即使两个正常 patch 内容有偏差，它们在投影空间中的距离
**仍然小于一个正常 patch 和异常 patch 的距离**。

合理的解释是：**模型在投影空间里学到了一个"异常方向"，所有异常 patch 都被推到这个
方向上，远离任何正常 patch**——不管这些正常 patch 是什么物体、什么光照、什么位置。
这个方向就是所谓的"通用异常先验"。


### 1.2 本实验目的

直接在 embedding 空间里验证："不同物体、不同位置的异常 patch 是否都被推到了同一个方向上？"
也就是：异常 patch 是否在特征空间里**跨物体跨位置地聚成一簇**，而不是各自分散在自己物体
的位置上？

## 二、实验设计

### 2.1 数据来源

testdata 的全部 721 个样本，每个样本含 `ref.png / query.png / gt.png`。

### 2.2 特征提取

对每个样本：

1. 用对比学习训练后的模型 (`encode_pair`) 提取 **query 图像**最后一层的 patch 特征 → contrastive
2. 用 baseline 模型 (`backbone` 直接输出，等价于 raw DINOv3，无 projector) → baseline
3. 把 `gt.png` 池化到 patch grid 分辨率（32×32），按阈值标记每个 patch：
   - `gt_pooled > 0.3` → **anomalous**
   - `gt_pooled < 0.05` → **normal**
   - 其他 → ignore（不进 UMAP）

### 2.3 类别不平衡处理

正常 patch 数量远多于异常 patch（异常区域往往 < 5% 像素）。如果直接全用，UMAP 会
被"巨量正常 patch"淹没。处理方法：

- **每类别独立子采样**：对每个类别分别采 200 个 anomalous patch + 200 个 normal patch
  （若该类异常 patch 不足 200 则全用）
- 实际收集：1168 anom + 1200 norm = **2368 个 patch**
- 6 个类别均权贡献，UMAP 形状不会被某类淹没

### 2.4 降维与可视化

- **UMAP**：`n_neighbors=15, min_dist=0.1, metric=cosine`
- 输出 4 张子图（2×2）：
  - 行 1：baseline / contrastive，按 **anomaly 状态**着色（红=异常，蓝=正常）
  - 行 2：baseline / contrastive，按**类别**着色，△=异常 marker，◯=正常 marker

### 2.5 量化指标（关键）

仅靠 UMAP 图直觉判断不够，配套 4 个指标定量回答"通用先验"假说：

| 指标 | 含义 | 假说预测 |
|------|------|---------|
| **Linear-probe AUROC** (5-fold CV) | embedding 上 logreg 区分 anom/norm 的 AUROC | — |
| **kNN purity (overall)** | 每个 patch 的 k=10 近邻里有多少同标签 | — |
| **kNN purity (anom only)** | 异常 patch 的 k 近邻里有多少也是异常 | — |
| **🔑 Cross-category anom neighbour rate** | 异常 patch 的 k 近邻里有多少是**其他类别**的异常 | **contrastive 应该 ≫ baseline** |

第 4 个是核心指标——它直接量化"不同物体的异常 patch 是否在特征空间里互相靠近"。

## 三、实验结果

### 3.1 量化指标（[metrics.json](outputs/dino3_vitb16_mydata_margin_1000_moredata/embedding/metrics.json)）

| 指标 | baseline | contrastive | Δ |
|------|----------|-------------|---|
| Linear-probe AUROC | **0.9799** ± 0.006 | 0.9035 ± 0.015 | **−0.076** ↓ |
| kNN purity (overall) | **0.9386** | 0.9179 | −0.021 ↓ |
| kNN purity (anom only) | **0.9699** | 0.9342 | −0.036 ↓ |
| kNN purity (norm only) | 0.9082 | 0.9020 | −0.006 ≈ |
| **Cross-cat anom nbr rate** | **0.00017** | **0.1596** | **+0.159** ↑↑ |

### 3.2 视觉形态（[embedding_comparison.png](outputs/dino3_vitb16_mydata_margin_1000_moredata/embedding/embedding_comparison.png)）

- **Baseline**：6 个类别各自抱团，△（异常）几乎贴着同色 ◯（同类正常），跨类别簇之间互不交叠
- **Contrastive**：类内仍有聚类，但 6 类的 △ 互相伸出长尾，向一个共同区域漂移；
  正常 patch 仍按类别聚集

## 四、结果分析

### 4.1 假说的核心证据：cross-cat anom neighbour rate 飙升约 800 倍

```
baseline:    0.00017   (10 个最近邻里几乎 0 个跨类异常)
contrastive: 0.15960   (10 个最近邻里平均 1.6 个是跨类异常)
```

**与随机期望对照**：

- 总样本中 5/6 比例是跨类，期望随机邻居比 ≈ 5×195 / (1168−1) ≈ **0.83**
- baseline 实际 = 期望的 **0.024%** → 异常 patch 几乎完全锁在本类内，跨类联系几乎为 0
- contrastive 实际 = 期望的 **19%** → 跨类联系大幅显现，但还远未完全融合

**结论**：训练**没有**把异常完全揉成跨类的一团（如果完全揉成一团，这个数应该 ≈ 0.83），
它做的是"在保留物体身份的同时，搭起异常 patch 之间的桥梁"。这是一种比"完全融合"
更精细的结果——既学到通用先验，又没破坏物体识别能力。

### 4.2 看似反向的两个指标，其实在讲同一个故事

Linear-probe AUROC 和 kNN purity (anom only) 在 contrastive 上略降，乍看像"特征变差"，
但实际是结构重组的副作用：

#### Linear-probe AUROC: 0.980 → 0.904 的解释
Baseline 上一个全局线性平面就能很好分开 normal/anom，因为每个类别内部的"异常方向"
本来就和别的类相似（阴影、突兀纹理偏向同一方向）。但 contrastive 的 256-d projection
把 anomaly 揉进一个**带弯曲的流形**——同一异常 patch 簇里横跨 6 个物体，分布形状
是球而不是平面，**线性平面变难找**。这是流形结构变化的副作用，不是特征质量下降。

#### kNN purity (anom only): 0.970 → 0.934 的解释
- Baseline：异常 patch 邻居 9.7/10 是异常（**且几乎全是同类**），purity 高
- Contrastive：异常 patch 邻居 9.34/10 是异常（其中 1.6 是**跨类**异常 + 7.74 同类异常 + 0.66 正常）

总 purity 略降，但**组成完全变了**：原来纯种内聚类，现在跨类伸手。这正是假说的预测，
**purity 略降反而是好事**——它说明"邻居池里掺进了别的物体的异常"。

### 4.3 一句话总结特征空间的变化

> **Baseline**：anomaly 各居其窝，每窝纯净。  
> **Contrastive**：anomaly 在每个窝里仍然有家，但**额外**和其他窝的 anomaly 串起来形成
> 一个跨物体的"暗网络"。

### 4.4 与下游任务的呼应

两个独立的实验互相印证：

| 测量 | baseline | contrastive | Δ |
|------|----------|-------------|---|
| Cross-cat anom nbr rate | 0.0002 | 0.1596 | +0.159 |
| Test image AUROC (macro) | 0.879 | 0.976 | +0.097 |

第二个是第一个的下游表现：训练在 embedding 空间建立了"跨类异常网络"，这个网络
在新物体（testdata 的 6 类）上提供了可迁移的判别力，体现为测试 AUROC 提升。

## 五、对未来实验的启示

### 5.1 数据 scaling 实验
在多个数据规模点（80 / 100 / 120 / 140 / 160 obj）训练，画一条
`cross_category_anom_neighbour_rate vs # training obj` 曲线。
如果单调上升，就直接量化"数据多样性 → 通用先验程度"的关系。

### 5.2 类间异常相似度矩阵
对 6 × 6 类别两两算 anomaly cluster 距离矩阵，看哪些类别的异常最像。
能揭示模型学到的"异常类型谱"。

### 5.3 跨类别异常 retrieval demo
拿 contrastive 空间里某个 Teapot 异常 patch 做 query，看它 top-10 的跨物体邻居都是什么。
如果都是真正的异常区域（视觉上各种"扭曲/突兀"），就有非常直观的"模型发现了通用异常"
的论文 figure。

### 5.4 极限位置
继续扩数据后，`cross_category_anom_neighbour_rate` 是否会从 0.16 进一步上升？
理论上限是 0.83（完全跨类融合），但更可能停在某个中间值，因为保留物体识别能力
也是有用的。这个停滞值可以揭示模型的"通用 vs 特异"平衡点。

```bash
conda activate RDD
cd /data2/baizeyu/RDD_CL

python scripts/visualize_embedding.py \
    outputs/dino3_vitb16_mydata_margin_1000_moredata/best.pth \
    --testdata-root testdata \
    --output outputs/dino3_vitb16_mydata_margin_1000_moredata/embedding \
    --per-cat-anom 200 --per-cat-norm 200
```

- `embedding_comparison.png` — 2×2 对比图
- `metrics.json` — 5 个量化指标
- `features.npz` — 原始 patch 特征（供进一步分析）
